Jenis Koneksi Mesin Fingerprint

A

SDK Resmi (Terbaik) — ZKTeco, Suprema, Nitgen

Mesin menyediakan SDK (library C++ / .NET / Python) yang bisa membaca log presensi secara langsung dari perangkat via jaringan LAN/WiFi. Middleware Python menggunakan SDK ini untuk polling data setiap beberapa menit.

B

Export File Otomatis (MDB/CSV)

Beberapa mesin bisa dikonfigurasi untuk menyimpan log ke file database (Access .mdb) atau CSV ke folder tertentu. Middleware memantau folder ini dengan watchdog dan upload perubahan ke Supabase.

C

PUSH Data via HTTP (Modern)

Mesin fingerprint tipe cloud (ZKTeco CloudX, dsb.) mendukung pengiriman data push ke URL endpoint. Buat endpoint kecil (dengan Express.js atau FastAPI) yang menerima data dan langsung masukkan ke Supabase.

D

Upload Manual CSV / Excel

Cara paling sederhana: export data dari software bawaan mesin (ZKTime, BioTime, dll.) ke CSV, lalu upload via fitur Import di halaman ini. Cocok jika tidak punya akses teknis ke mesin.

1. Buat Project di Supabase

1

Daftar & Buat Project

Buka supabase.com → New Project → isi nama, password database, pilih region terdekat (Singapore).

2

Buat Tabel di SQL Editor

Buka SQL Editor di Supabase, jalankan query berikut:

-- Tabel karyawan CREATE TABLE karyawan ( id UUID PRIMARY KEY DEFAULT gen_random_uuid(), fp_id TEXT UNIQUE NOT NULL, -- ID di mesin fingerprint nama TEXT NOT NULL, divisi TEXT, jabatan TEXT, gaji_pokok NUMERIC DEFAULT 0, aktif BOOLEAN DEFAULT true, created_at TIMESTAMPTZ DEFAULT now() ); -- Tabel absensi CREATE TABLE absensi ( id UUID PRIMARY KEY DEFAULT gen_random_uuid(), fp_id TEXT REFERENCES karyawan(fp_id), nama TEXT, tanggal DATE NOT NULL, check_in TIME, check_out TIME, total_jam NUMERIC, is_weekend BOOLEAN, poin_pokok NUMERIC, poin_lembur NUMERIC, total_poin NUMERIC, sumber TEXT DEFAULT 'manual', -- 'fingerprint' atau 'manual' created_at TIMESTAMPTZ DEFAULT now() ); -- Index untuk performa CREATE INDEX idx_absensi_tanggal ON absensi(tanggal); CREATE INDEX idx_absensi_fpid ON absensi(fp_id);
3

Aktifkan Row Level Security (RLS)

Di Supabase → Authentication → Policies. Tambahkan policy untuk membatasi akses. Untuk akses publik dari HTML ini, gunakan anon key dan buat policy select:

ALTER TABLE absensi ENABLE ROW LEVEL SECURITY; ALTER TABLE karyawan ENABLE ROW LEVEL SECURITY; -- Izinkan baca untuk semua (sesuaikan dengan kebutuhan) CREATE POLICY "Allow read" ON absensi FOR SELECT USING (true); -- Izinkan insert dari middleware (service_role key) CREATE POLICY "Allow insert" ON absensi FOR INSERT WITH CHECK (true);
4

Ambil API Keys

Buka Project Settings → API. Catat:
• Project URL: https://xxxxx.supabase.co
• anon public key: untuk frontend (HTML ini)
• service_role key: untuk middleware (JANGAN expose ke frontend!)

Middleware Python — Baca Fingerprint → Supabase

Jalankan script ini di komputer yang terhubung ke mesin fingerprint (LAN yang sama).

1

Install Dependencies

pip install supabase pyzk requests schedule
2

Konfigurasi (config.py)

# config.py SUPABASE_URL = "https://xxxxx.supabase.co" SUPABASE_SERVICE_KEY = "eyJ..." # service_role key FINGERPRINT_IP = "192.168.1.100" # IP mesin fingerprint FINGERPRINT_PORT = 4370 # port default ZKTeco POLL_INTERVAL_MINUTES = 5 # cek setiap 5 menit JAM_KERJA_STANDAR = 9 # jam NILAI_POKOK_WEEKDAY = 1 NILAI_POKOK_WEEKEND = 2 UNIT_LEMBUR = 0.5 # poin per 0.5 jam
3

Script Utama (middleware.py)

import zk, schedule, time, math from datetime import datetime, date from supabase import create_client from config import * from collections import defaultdict sb = create_client(SUPABASE_URL, SUPABASE_SERVICE_KEY) def hitung_poin(total_jam, is_weekend): """Hitung poin berdasarkan jam kerja""" poin_pokok = NILAI_POKOK_WEEKEND if is_weekend else NILAI_POKOK_WEEKDAY selisih = max(0, total_jam - JAM_KERJA_STANDAR) poin_lembur = math.floor(selisih / UNIT_LEMBUR) * UNIT_LEMBUR return round(poin_pokok, 2), round(poin_lembur, 2) def sync_fingerprint(): conn = zk.ZK(FINGERPRINT_IP, port=FINGERPRINT_PORT, timeout=5) zk_instance = conn.connect() attendances = zk_instance.get_attendance() zk_instance.disconnect() # Group by (user_id, date) by_day = defaultdict(list) for att in attendances: key = (att.user_id, att.timestamp.date()) by_day[key].append(att.timestamp) for (fp_id, tgl), times in by_day.items(): times.sort() check_in = times[0] check_out = times[-1] total_menit = (check_out - check_in).total_seconds() / 3600 is_we = tgl.weekday() >= 5 p_pokok, p_lembur = hitung_poin(total_menit, is_we) data = { "fp_id": str(fp_id), "tanggal": str(tgl), "check_in": check_in.strftime("%H:%M"), "check_out": check_out.strftime("%H:%M"), "total_jam": round(total_menit, 2), "is_weekend": is_we, "poin_pokok": p_pokok, "poin_lembur": p_lembur, "total_poin": p_pokok + p_lembur, "sumber": "fingerprint" } sb.table("absensi").upsert( data, on_conflict="fp_id,tanggal").execute() print(f"[{datetime.now()}] Sync selesai: {len(by_day)} record") schedule.every(POLL_INTERVAL_MINUTES).minutes.do(sync_fingerprint) sync_fingerprint() # langsung jalankan pertama kali while True: schedule.run_pending() time.sleep(30)
⚠️ Script SDK di bawah ini menggunakan library pyzk yang kompatibel dengan mesin ZKTeco. Untuk merek lain, gunakan SDK resminya masing-masing.
